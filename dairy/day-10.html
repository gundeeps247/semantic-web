Day 10

SPARQL: A query language for databases, able to retrieve and manipulate data stored in Resource Description Framework (RDF) format.

Query: A request for data or information from a database.
Data set too big: Handling large datasets can be challenging with SPARQL queries.

We want to represent database using SPARQL: SPARQL is used to represent and query data in RDF format.

4 key things:

Prefix URI (urn, etc.): URIs are used to uniquely identify resources in RDF data.

Select - to fetch: The SELECT clause is used to specify the variables to be returned by the query.

From: The FROM clause specifies the data source for the query.

Where: The WHERE clause contains the conditions that must be met for data to be returned.

Tool (Apache Jena Fuseki): Apache Jena Fuseki is a SPARQL server used for managing RDF data.

Will use for query execution: Apache Jena Fuseki will be used to execute SPARQL queries.

Fetch XML file and generate queries: SPARQL can fetch data in XML format, and queries can be generated to retrieve specific data.

Commands used:
Data: Commands to interact with the data in the RDF store.
Install: Steps to install Apache Jena Fuseki.
Import data: Loading data into the RDF store.
Normal command (limit 10): Can change the result limit and paginate data.
RDF Triple: Subject, predicate, and object form the RDF data structure.
